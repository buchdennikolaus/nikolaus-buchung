# Project K - Nikolaus Booking System Prototype

**Status**: Active  
**Date Started**: 2026-04-24  
**Project Owner**: Timothy Kaufmann  

## Overview

A clickable front-end prototype for a web-based booking system for Nikolaus/Krampus visits organized by a Kolping family. This is a UI/UX demonstration tool for stakeholder presentation, not a production system.

## Objectives

- Create a fully clickable front-end prototype demonstrating the booking flow
- Simulate user registration, slot selection, and booking confirmation
- Provide an admin overview interface
- Follow Kolping design guidelines (Orange #f05a00, Black, White)
- Mobile-first responsive design

## Technologies

- HTML5
- CSS3 (Mobile-first, responsive)
- Vanilla JavaScript (no frameworks needed for prototype)
- Mock data simulation

## Key Features

### Public User Flow
1. Landing page with event information
2. Family data form with dynamic duration calculation
3. Time slot selection (Dec 5-6, 17:00-20:00)
4. Booking confirmation
5. Success page with summary

### Admin Dashboard
- View all bookings
- Filter by date
- Assign teams
- Track document status
- Internal notes

## Design Guidelines

- **Primary Color**: Orange (#f05a00) - buttons, highlights, active states
- **Secondary Color**: Black/Dark grey - text, borders
- **Background**: White
- **Style**: Clean, minimal, professional (no Christmas kitsch)
- **Typography**: Sans-serif, large spacing, clear sections
